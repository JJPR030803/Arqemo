## Implemented: `arqemo-cli/src/main.rs`

The CLI is a thin routing layer using `clap` with the following subcommands:

| Command | Arguments | Description |
|---------|-----------|-------------|
| `apply` | `theme`, `--dry-run` | Apply a theme to the desktop |
| `validate` | `theme`, `--info` | Validate a theme file without applying |
| `list` | `--complete-path` | List all available themes |
| `init` | — | Initialise the arqemo directory structure |

All commands delegate to `arqemo_core` — no logic lives in the CLI crate.

## Implemented: `arqemo-core/src/lib.rs`

### `init() -> Result<()>`

Creates the full arqemo directory structure:
- `~/.config/arqemo/`
- `~/.config/arqemo/themes/`
- `~/.config/arqemo/templates/`
- `~/.cache/arqemo/`
- `~/.cache/arqemo/rendered/`
- `~/.cache/arqemo/themes/`

Idempotent — safe to run multiple times.

### `list_themes(complete: bool) -> Result<()>`

Lists all themes in `~/.config/arqemo/themes/`.

- `complete: false` — prints theme name only
- `complete: true` — prints full path

### `validate_theme(theme: &str, info: bool) -> Result<()>` *(new)*

Validates a theme without applying it. Two-stage validation:

1. **File validation** (`validate::file::validate_file`) — parses `theme.toml` and checks structure
2. **Semantic validation** (`validate::semantic::validate_semantic`) — checks logical correctness

Arguments:
- `theme` — name of the theme directory (must exist in `~/.config/arqemo/themes/`)
- `info` — if `true`, prints success message and file path

Errors if:
- Theme directory does not exist
- `theme.toml` is missing
- File validation fails (malformed TOML, schema mismatch)
- Semantic validation fails (invalid references, constraint violations)
