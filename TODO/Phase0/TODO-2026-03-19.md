## Summary

Fixed wallpaper activation pipeline and updated synthwave theme to use
pool-based wallpaper selection with swww transition config.

## Fixed: `arqemo-core/src/apply/wallpaper.rs`

The `activate_image()` function was broken — `arqemo apply synthwave` failed
with `wallpaper command failed: No such file or directory (os error 2)`.

### Root causes (all fixed)

| Bug | Fix |
|-----|-----|
| Tilde `~` in wallpaper path not expanded | Added `expand_tilde()` — resolves `~` via `$HOME` before passing to commands |
| `backend` field ignored — code always tried hyprpaper → swww → swaybg cascade | Now reads `config.wallpaper.backend` and dispatches directly to the correct tool |
| `hyprpaper` invoked as CLI binary with `--set_wallpaper` flag (wrong interface) | Fixed to use `hyprctl hyprpaper preload` + `hyprctl hyprpaper wallpaper` (IPC protocol) |
| `[wallpaper.transition]` config ignored for swww | Now passes `--transition-type`, `--transition-duration`, `--transition-fps`, `--transition-bezier` |
| Final fallback to `swaybg` (not installed) produced the ENOENT | Removed silent cascade — backend is explicit, not a guessing game |

### New function signatures

- `activate_image(config)` — reads `backend` field, defaults to `Hyprpaper` when `None`
- `try_hyprpaper(path)` — unload all → preload → wallpaper via `hyprctl hyprpaper`
- `try_swww(path, transition)` — `swww img` with optional transition args
- `expand_tilde(path)` — `~/...` → `/home/<user>/...`

## Updated: `themes/synthwave/theme.toml`

Wallpaper section migrated from single `path` to pool-based config:

**Before:**
```toml
[wallpaper]
mode    = "image"
backend = "swww"
path    = "~/.config/arqemo/themes/synthwave/wallpapers/neonlit-synthwave-cityscape.png"
```

**After:**
```toml
[wallpaper]
mode    = "image"
backend = "swww"
pool    = "~/.config/arqemo/themes/synthwave/wallpapers"
default = "neonlit-synthwave-cityscape.png"

[wallpaper.transition]
type     = "wipe"
duration = 1.5
fps      = 60
```

Pool contains 10 images for runtime switching via `arqemo wallpaper set/next/random/reset`.
